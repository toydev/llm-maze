## リポジトリ構成の再編成

迷路実験が一段落したら、関連実験を1リポジトリでまとめる構成に移行する。

```
/（リポジトリルート）
├── maze/           # 迷路実験
│   ├── mazes/
│   ├── src/
│   └── output/
├── coords/         # 座標計算実験（新規）
│   ├── src/
│   └── output/
├── shared/         # 共通コード（LLM呼び出し等）
└── docs/           # 全体のドキュメント
```

**メリット**:
- LLM呼び出し周りの共通コードを再利用
- プロンプト戦略の知見を横断的に適用
- 実験結果を比較しやすい

## 座標計算（相対位置指定）実験

1年半前の実験で「相対位置の指定が苦手」という結果があった。今回の知見を活かして再実験する。

**仮説**:
- graph戦略のように「関係」として明示的に展開すれば改善する可能性
- 例: 「北に3マス」→「北隣接ノード→その北隣接→その北隣接」と展開

**実験内容案**:
- 相対方向指定（北に2歩、東に3歩など）
- 座標計算（現在地から目標地点への相対位置）
- 方角認識（目標は現在地の北東にある、など）

mazeプロジェクトの知見をDQAIに適用する。
