# プロンプト戦略比較実験

## 概要

gpt-oss:20b（Ollama ローカル実行）で各プロンプト戦略の精度と速度を比較。

## 結論

**graph 戦略が最適解**。速度・精度ともに最高。

## 実験条件

- モデル: ollama:gpt-oss:20b
- 迷路: 5x5_simple（7ポジション）、10x10_complex（25ポジション）
- 評価基準: 「遠ざからない移動」を正解とする

## 結果

### 10x10_complex

| 戦略 | 正解率 | 時間 | 備考 |
|------|--------|------|------|
| graph | 100% (25/25) | 6:45 | **最速・最高精度** |
| list | 100% (25/25) | 12:31 | graph の約2倍 |
| matrix-sep | 96% (24/25) | 53:37 | 遅い |
| simple | 92% (23/25) | 76:17 | 最も遅い |
| matrix-embed | 88% (22/25) | 50:34 | 精度も低め |

### 5x5_simple

| 戦略 | 正解率 | 時間 |
|------|--------|------|
| graph | 100% (7/7) | 1:09 |
| list | 100% (7/7) | 1:23 |
| matrix-sep | 100% (7/7) | 1:40 |
| matrix-embed | 100% (7/7) | 2:53 |

## 戦略の特徴

### graph（隣接リスト）
- 各ノードの隣接関係を明示的に列挙
- LLM が「関係のグラフ」として処理しやすい
- 2D空間認識の負担を軽減

### list（座標リスト）
- 歩行可能な座標のリストのみ
- 隣接関係は LLM に推測させる
- シンプルだが graph より遅い

### matrix-sep（位置分離型行列）
- 0/1 の2D行列 + 位置情報を別途記載
- 行列解釈に時間がかかる

### matrix-embed（位置埋込型行列）
- S/E/C を行列内に埋め込む
- 情報が混在し、精度・速度ともに劣る

### simple（テキスト描画）
- 迷路をそのまま文字列で表現
- 直感的だが処理が最も遅い

## 考察

### graph が強い理由
- 隣接関係の明示により「空間」を「関係」に変換
- LLM のテキスト処理能力を活かせる
- トークン数が少なく、推論負荷も低い

### 注意点
- 10x10_complex は一方通行に近い構造
- history（訪問履歴）を渡しているため、「隣接＋未訪問」で答えが絞られる可能性
- 真の空間認識能力は 10x10_branch（分岐あり）で要検証
