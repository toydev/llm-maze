# AGENTS.md

このプロジェクトの目的は、LLM (大規模言語モデル) を使った迷路探索の試行錯誤をすることです。最終的なゴールは、迷路探索の可能な方法を見つけることです。そのための適切な表現方法の探求を行います。

対象となる迷路は、格子状のマス目を持った2D迷路です。各マス目には移動の可否が設定されており、迷路には開始点と終了点が存在します。

アプローチとしては、まずサイズの小さい簡単なケースから開始し、そこから徐々に複雑なケースへと発展させていく反復的な方法を取ります。

## 具体的な実装方針

- **迷路の表現**: 迷路の構造はテキスト形式（例：`#`を壁、`S`を開始点、`E`を終了点とする）で定義・設定できるようにします。
- **中心となるタスク**:
  1. テキスト形式の迷路を入力として受け取ります。
  2. 入力された迷路情報から、LLMに対するプロンプトを動的に生成します。
  3. 生成したプロンプトをLLMに与え、開始点`S`から終了点`E`までの経路を回答させます。
- **試行錯誤のアプローチ**: プロンプトの表現方法や、LLMに期待する出力フォーマット（例：座標のリスト、移動方向の連続指示など）について、複数のバリエーションを試し、最適な方法を探求します。
- **正解判定**: LLMが生成した経路が正しいかどうかを自動で検証するため、A*（A-Star）探索などのアルゴリズムを用いた、従来型の迷路探索プログラムも別途実装します。
